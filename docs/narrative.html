06 · Under the hood

Built on the real Even Hub SDK.
Not a mockup. A plugin.

Runtime

The plugin shell

  • TypeScript + Vite. Modern toolchain, fast iteration.
  • Runs inside the Even Realities App WebView.
  • Uses @evenrealities/even_hub_sdk.
  • Ships as a packaged .ehpk plugin.

Render

Pixels to the glasses

  • Scenes composed at 576 × 144, tiled to both eyes.
  • Pushed as greyscale image buffers over BLE.
  • Five bespoke scenes: overview + four turn steps.
  • Readable type, clear POI badges, route hierarchy.

Interaction

Hands stay off the phone

  • Click · confirm and advance.
  • Double click · go back / home.
  • Swipe up / down · next / previous step.
  • Temple touchpad primary, R1 ring optional.
07 · Honest scope

What shipped this sprint, and what did not.

In the box

  • Full on-glass flow: home → FROM → TO → overview → steps → arrived.
  • MIA Terminal D with curated gates, lounges, food POIs.
  • One real route: D30 → Starbucks.
  • Five hand-composed Wayfinder-style scenes.
  • Runs in the Even Hub simulator and on real glasses.
  • Graceful fallback for unsupported FROM / TO pairs.

Not yet · deliberately

  • Live indoor positioning.
  • Real Wayfinder SDK route stream.
  • Multi-airport or multi-venue support.
  • Real venue data ingestion pipeline.
  • Account, backend, analytics, ops.
  • Production polish and accessibility pass.
